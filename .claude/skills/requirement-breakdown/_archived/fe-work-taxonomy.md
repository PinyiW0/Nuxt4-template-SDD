# 前端工項分類法（六層 checklist）

拆解時，每個介面都用這六層掃一遍：命中就把工項濃縮進該介面的「說明」。目的是**一致、不漏項**——尤其影音/視覺化、即時連線、第三方整合這些容易被低估的層。

> 用法：Phase 2 對每個介面逐層問「這介面有沒有這層的工？」。跨介面共用的（設計系統、RWD、效能、測試）依「跨切面工項」段決定獨立列或併係數。

---

## L1 畫面 / UI

最基本的畫面與互動骨架。

- 頁面骨架與路由（layout、navigation、頁面切換）
- 共用元件（按鈕、卡片、表單控件、modal、toast、空狀態、錯誤狀態）
- 表單與驗證（輸入、即時驗證、送出回饋）
- 列表與詳情（table / card / list、分頁、排序、篩選）
- loading / empty / error 三態

## L2 影音 / 視覺化（這領域的重頭，最易低估）

涉及影像、播放、座標、圖形的工，難度與工時都偏高。

- 影音播放器：一般播放、**慢動作 / 逐幀 / 變速**、時間軸 scrub、多路同步
- 即時影像串流播放：低延遲、播放器掛載與自救（搭 `streaming` skill 知識）
- **WebRTC 即時監看**：B 端後台接攝影機即時畫面（取景確認、運作狀態、多路）——即時影像 **不是只有 C 端播放**，後台監看也算（搭 `realtime` / `streaming` skill）
- 圖形 / 座標視覺化：球場平面圖、**落點 / 軌跡標註**、座標映射（canvas / svg）
- 影像編輯類介面：裁切框、縮放、拖拽擺放、template（複雜編輯器 = 高成本）
- 動效 / 轉場 / 特效呈現（前端呈現面，非演算法合成）

## L3 資料 / 狀態

資料怎麼進畫面、狀態怎麼管。

- API 串接 client 層（查詢、CRUD、上傳 / 下載、檔案處理）
- **即時連線**：WebSocket / SSE 推播（得分事件、系統狀態、通知）；連線生命週期、重連（搭 `realtime` skill 知識）
- 狀態管理（store 設計、跨頁共享、快取）
- 認證 / 授權（登入、token、角色分權的前端守門）

## L4 流程 / 整合

跨系統、跨服務的串接流程（前端只估畫面與串接 client）。

- LINE 生態：**先分前後端歸屬**——只有 **LIFF 網頁**（HTML/JS 畫面）才估前端；**Bot 對話流**（rich menu 選單觸發、輸入 Email/手機、OTP 驗證碼問答、狀態機綁定）是 **LINE 平台 + 後端**，前端工項通常為零。推送：發送在後端，前端只在 LIFF 內呈現
- 分享流程：QRCode 產生 / 掃描、深連結、Email / 連結分享落地頁
- 金流 / 會員：第三方支付前端串接、會員綁定與分級的畫面
- 報到 / 啟動類流程：掃碼報到、裝置綁定、遠端啟動的操作面

## L5 後台 / 管理（B 端）

管理者端常見模組。

- **後台基礎建設（B 端案必估的起手工）**：後台 layout / UI 風格、**權限架構（RBAC 前端骨架）**、**JWT 驗證架構**（登入態、路由守衛、token 刷新）——任何後台的地基，獨立列、別漏
- 帳號管理：登入、增刪改、角色 / 權限分級的介面（RBAC 前端面）
- 監控 / 儀表板：系統狀態、遠端啟動 / 監控、異常告警呈現、Dashboard 數據圖表
- 資料管理：影像 / 紀錄列表、檢索、刪除、標註、批次操作
- 內容管理：CMS 式編輯、template 管理

## L6 非功能 / 工程（常為跨介面共用）

- **UIUX 設計**：wireframe、互動規格、視覺稿（資深前端要把設計工估進去）
- 效能：影音 / 首屏優化，達成硬 KPI（如「喚醒 < 3s」「長邊 1080p@30fps」）
- 多端 / RWD 適配
- E2E 測試
- 整合 / 部署的前端配合工

## 跨切面工項：獨立列 vs 併係數

| 工項 | 處理方式 | 依據（公司歷史） |
|------|----------|------------------|
| 設計系統 / 共用元件 | **獨立列**一次（不要每介面重複算） | 多案把共用元件獨立估 |
| RWD 多端適配 | **乘數** ×1.25–1.5，或整包獨立列 | 左訓 RWD ×1.5、青青 RWD 獨立 22 人天 |
| 效能優化（硬 KPI） | 對相關介面**加成**到悲觀端 | 即時 / 高畫質 KPI 拉高難度 |
| E2E 測試 | **獨立列**（前端佔比） | 多案測試獨立列 11–33 人天 |
| 整合 / 部署 | **獨立列**（前端佔比 2–5 人天） | 國泰前端部署 2–4、華山部署前端佔比 |

## 領域訊號 → 自動帶出的高成本工項

看到這些需求字眼，對應層幾乎必然出現，別漏：

| 需求訊號 | 帶出工項（層） | 風險 |
|----------|----------------|------|
| 慢動作 / 壓線 / 逐幀回放 | 變速播放器 + 時間軸（L2） | 高 |
| 落點 / 軌跡 / 平面圖 | 座標映射視覺化 canvas（L2） | 高 |
| 即時 / 推播 / 得分事件 | WebSocket / SSE（L3） | 中–高 |
| LINE LIFF 網頁 | LIFF 頁面 + 推送呈現（L4，**前端**） | 中–高 |
| LINE 綁定 / OTP / 選單問答 | **Bot 對話流＝後端**，前端通常為零（L4） | 前端零 |
| QRCode 產生（後台） | 後台生成 QRcode 畫面（L5）；掃碼端多為 LINE / 相機，非前端 | 中 / 低 |
| 帳號 / 角色 / 後台 | 後台基礎建設 + 帳號管理 + RBAC（L5） | 中–高 |
| 攝影機 / 監看 / 即時畫面 | **WebRTC 即時監看（L2）** | 高 |
| 啟動 / 監控 / 異常重啟 | 監控儀表板（L5） | 中 |
| 喚醒 < Ns / 長邊 1080p | 效能優化（L6） | 中–高 |
| 無需安裝 APP | 載體假設：web / PWA / LIFF（影響全盤） | — |
