## CSV 區塊（可貼 Excel）

報告末附一段 fenced CSV，欄位對齊公司表頭，方便直接貼進「研發部預估工時」：

```csv
子系統,端,項目,說明,屬性,難易度,樂觀人天,悲觀人天,人月(悲觀),備註
{子系統},C,{介面},{工項},前端,高,5,8,0.36,RAG:中信
...
```

`屬性` 一律填「前端」（全端工項已切前端佔比）。`人月(悲觀)` = 悲觀人天 ÷ 22，取兩位小數。

---

## 規則

- 每個子系統一個小節 + 小計；最後總計。
- 每列**必有**難易度與備註（錨點來源 `RAG:案名` 或 `內建表`）。
- 數字一律區間（樂觀 / 悲觀），不給單點。
- **0. User Flow & User Story** 為 Phase A 對齊產物，置於最前、**不可省**（單純案可精簡帶過）。
- 排除清單、假設、待確認三段**不可省**——這是 rough spec 評估的可信度來源。
- 結尾固定重申「不含溝通成本與需求定義」。
